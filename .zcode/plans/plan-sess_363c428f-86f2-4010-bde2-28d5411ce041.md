# Plan : Système « Donner un coup de pouce » + Page des requêtes (style WhatsApp)

## Vue d'ensemble
Transformer la section « 💛 Soutenir le projet » (et le header) en un bouton **« Donner un coup de pouce »**. Au clic → instructions détaillées → formulaire de requête classée en 3 catégories avec don associé. Chaque requête envoyée apparaît sur une **page publique de suivi** avec des bulles style WhatsApp et **deux accusés de réception** (check développeur + check utilisateur), triés par catégorie puis par statut.

## Choix validés par l'utilisateur
- **Accès page** : publique en lecture, mais les checks sont **protégés par codes** (dev = code secret ; user = code unique remis après envoi).
- **Montant** : montant numérique **+ référence de transaction** (texte).
- **2e check utilisateur** : **code unique** généré + **localStorage** + lien « Suivre les requêtes ».

## 1. Backend

### 1a. Nouveau modèle `models/Request.js`
```
- category      : 'bug' | 'ux' | 'tech'   (requis)
- message       : String                   (requis)
- amount        : Number                   (requis)
- network       : String                   (M-Pesa/Airtel/Orange/Africell)
- transactionRef: String                   (référence/ID de transaction)
- amountOk      : Boolean                  (montant ≥ minimum de la catégorie, calculé au post)
- userCode      : String                   (code unique 6 char, généré serveur, pour le 2e check)
- devCheck      : Boolean (false) + devCheckAt: Date
- userCheck     : Boolean (false) + userCheckAt: Date
- createdAt     : Date
```

### 1b. Nouvelles routes API (dans `server.js`, style existant)
- **`POST /api/requests`** — crée une requête. Reçoit `{ category, message, amount, network, transactionRef }`. Sert à :
  - valider que `amount` ≥ minimum de la catégorie (1 / 5 / 10 $), sinon erreur 400 ;
  - calculer et stocker `amountOk` ;
  - générer un `userCode` unique (6 caractères alphanum, vérif d'unicité en BD) ;
  - renvoyer `{ userCode, id }`.
- **`GET /api/requests`** — renvoie les requêtes **triées** : (1) bug non confirmés, (2) UX non confirmés, (3) tech non confirmés, (4) confirmés (les deux checks cochés). Au sein de chaque groupe : du plus ancien au plus récent (file d'attente).
- **`POST /api/requests/:id/dev-check`** — bascule `devCheck`. Body `{ devCode }`, vérifié contre `process.env.DEV_CODE`. Renvoie 403 si faux.
- **`POST /api/requests/:id/user-check`** — bascule `userCheck`. Body `{ userCode }`, vérifié contre le `userCode` de la requête. Renvoie 403 si faux.

### 1c. `.env`
Ajouter une variable `DEV_CODE` (je proposerai une valeur aléatoire lisible, ex. `PARABOLE-2026`).

## 3. Frontend — Nouvelle page `public/requests.html` + `public/requests.js`

Page publique de suivi, style **fil de messages WhatsApp** :
- En-tête « Requêtes de la communauté » + 4 sections (Bug · UX · Tech · Confirmés).
- Chaque requête = **bulle** (gris clair, alignée gauche) avec :
  - **Badge catégorie** coloré + montant + réseau.
  - Le **message** (texte).
  - Ligne « Réf. transaction : … » + pastille « Don joint ✓ » / « Don insuffisant ✗ ».
  - Horodatage.
  - **Deux checks** en bas à droite (style accusés WhatsApp) :
    - `✓✓` gris = seul le check dev est coché ;
    - `✓✓` bleu = les deux cochés (confirmé) ;
    - un seul `✓` = aucun check encore.
  - Au survol/clic sur un check : **prompt du code** correspondant (dev ou user) → appel API. Si valide, le check se met à jour.

`requests.js` : fetch `GET /api/requests`, rendu des bulles, gestion des clicks de check avec prompts de code.

## 5. Texte des instructions (rédigé, à afficher dans le modal)
> **Voici comment nous soutenir activement :**
> 1. **Décrivez la modification** que vous aimeriez voir sur le site (un bug, une amélioration, une nouvelle fonctionnalité technique).
> 2. **Choisissez la nature** de votre demande parmi les 3 catégories ci-dessous — chacune fixe le don minimum.
> 3. **Effectuez votre don** via l'un des réseaux (M-Pesa, Airtel, Orange, Africell) au numéro affiché, pour un montant au moins égal au minimum de la catégorie choisie.
> 4. **Indiquez le montant envoyé et la référence** de la transaction (le numéro/ID de confirmation reçu par SMS).
> 5. **Envoyez votre requête.** Un **code unique** s'affiche : **conservez-le**, il vous permettra plus tard de confirmer que la modification a été vue et faite.
>
> Une fois envoyée, votre requête apparaît sur la page de suivi. Dès que le développeur réalise la modification, le 1er accusé ✓✓ s'active. Quand vous constatez le changement, utilisez votre code pour activer le 2e accusé ✓✓ (bleu) = demande confirmée.
